# SDLC

## Objective

Establish a predictable delivery path for `[PROJECT_NAME]` that keeps core correctness, orchestration behavior, and UI usability decoupled.

## Definition of Ready

- Work item exists in `docs/backlog.md`
- Acceptance criteria are clear
- Affected architecture boundary is identified
- Dependencies and assumptions are documented

## Definition of Done

- Code matches the relevant architecture boundary
- Behavior satisfies acceptance criteria
- Tests cover new rules or decision logic when applicable
- Spec and backlog remain accurate
- Review evidence includes a direct path to inspect or run the result
